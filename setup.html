Beer Festival App
Setup Guide

Everything you need to get the festival app live — from Firebase setup to handing over to the festival team. Should take about 20 minutes.

What You Have
📱
index.html
The public-facing app. Festival attendees use this on their phones to browse beers, save a list, and see live stock status.
🔧
admin.html
The staff portal. Admins configure the festival, import beers and manage vendors. Vendors log in here to update their stock levels.
⚙️
config.js
The only file you need to edit. Paste your Firebase project credentials here once and you're done.

Step 1 — Create a Firebase Project
1
Go to Firebase Console
Visit console.firebase.google.com and sign in with a Google account. Click Add project, give it a name (e.g. norwich-craft-beer-fest-2026), and click through to create it. You don't need Google Analytics.
2
Enable Firestore Database
In the left sidebar, click Build → Firestore Database → Create database. Choose Start in production mode. Pick the closest region (europe-west2 for UK). Click Enable.
3
Enable Authentication
In the sidebar, click Build → Authentication → Get started. Under Sign-in method, enable Email/Password. Save.
4
Get your config and paste it into config.js
Go to Project Settings (the gear icon) → General → scroll down to Your apps → click the web icon (</>) → Register app → copy the firebaseConfig object. Open config.js and replace the placeholder values:
// config.js const FIREBASE_CONFIG = { apiKey: "AIzaSyAbc123...", authDomain: "your-project.firebaseapp.com", projectId: "your-project", storageBucket: "your-project.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc..." };
Step 2 — Set Firestore Security Rules
5
Paste the security rules
In Firestore, click the Rules tab. Replace everything with the rules below and click Publish. These ensure the public app can read beers, but only authorised staff can make changes.
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /settings/{doc} { allow read: if true; allow write: if isAdmin(); } match /beers/{beerId} { allow read: if true; allow create, delete: if isAdmin(); allow update: if isAdmin() || isVendorOf(beerId); } match /users/{uid} { allow read: if request.auth.uid == uid || isAdmin(); allow write: if isAdmin(); allow create: if request.auth.uid == uid; } function isAdmin() { return request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } function isVendorOf(beerId) { return request.auth != null && get(/databases/$(database)/documents/beers/$(beerId)).data.vendorUid == request.auth.uid; } } }
Step 3 — Create the First Admin Account
6
Create the admin user in Firebase Authentication
Go to Authentication → Users → Add user. Enter the admin's email and a strong password. Click Add User. Copy the UID shown in the user list (looks like: abc123XYZ789…).
7
Create their user document in Firestore
Go to Firestore → Start collection. Collection ID: users. Document ID: paste the UID from step 6. Add these three fields:
role (string) → "admin" email (string) → "admin@festival.com" displayName (string) → "Festival Admin"
That's the only manual Firestore entry you'll ever need to make. All future users are created through the admin portal itself.
Step 4 — Deploy to Netlify
8
Deploy the three files
Go to netlify.com, sign up free, and drag the whole project folder onto the deploy area. Done. You'll get a URL like https://your-festival.netlify.app.

To use a custom domain (e.g. app.norwichcraftbeerfest.co.uk), go to Site settings → Domain management and follow the instructions.
You're live. The public app is at /index.html and the staff portal is at /admin.html. Share the public URL with attendees and the admin URL with the festival team.
Step 5 — Festival Setup (Admin does this)
9
Configure the festival look and feel
The admin logs into /admin.html, goes to Settings, and sets the festival name, tagline, logo, and colour scheme. Changes go live to the public app immediately.
10
Import the beer list
Go to Import. Three options:

CSV — easiest for a spreadsheet. Columns: brewery, name, style, abv, description, tags, vendor_email

Image — photograph or scan the printed beer list and upload it. Requires a Claude API key (get one free at console.anthropic.com). The AI extracts all the beers automatically.

Paste text — copy text from a PDF or type it in. One beer per line, format: Brewery – Name – Style – ABV%
11
Add vendor accounts for each brewery
Go to Vendors → Add Vendor. Enter the brewery's name, email, a temporary password, and which brewery they manage. They'll log into /admin.html with those credentials and only see their own beers.
12
Assign beers to vendors
In the Beers section, click Edit on any beer and set the Assigned Vendor dropdown. Or, include a vendor_email column in your CSV import and it's done automatically.
On the Day
For vendors: Send them the link to /admin.html and their email + password. On their phone, they'll see only their beers and a simple dropdown: 🟢 Available → 🟡 Low Stock → 🔴 Sold Out. That's it.
For attendees: Share the link to /index.html. Works on any smartphone browser, no app download needed. They can save a tasting list and see stock status update in real time as bars sell out.
Reusing next year: Before the festival, delete all beers from the Beers section (or use a new Firebase project) and re-import the new year's list. The colours, name and vendor accounts will all still be there.
